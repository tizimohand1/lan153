3.1: Security Implications of Different Architecture Models

Learning Objectives

Students will be able to:

  • Analyze security implications of cloud, edge, and fog computing architectures
  • Evaluate the security considerations of virtualization and containerization
  • Assess architecture models based on availability, cost, scalability, and risk factors
  • Compare centralized vs. decentralized architecture security implications

Key Terms

  • Cloud Computing: On-demand delivery of computing services over the internet
  • Edge Computing: Computing performed at or near the data source rather than in centralized cloud
  • Fog Computing: Decentralized computing infrastructure extending cloud capabilities to edge devices
  • Thin Client: Lightweight computer dependent on server for processing
  • Containerization: OS-level virtualization for deploying applications in isolated environments
  • Microservices: Architectural approach using small, independent services
  • Infrastructure as Code (IaC): Managing infrastructure through machine-readable definition files
  • Serverless: Cloud computing model where cloud provider manages server infrastructure

Key Concepts

Architecture and Infrastructure Models

  • Cloud Computing Models: Shared responsibility matrix, hybrid considerations, third-party vendors
  • Infrastructure Approaches: IaC, serverless computing, microservices architecture
  • Network Infrastructure: Physical isolation, logical segmentation, software-defined networking
  • Computing Models: On-premises, centralized vs. decentralized, containerization, virtualization
  • Specialized Systems: IoT devices, ICS/SCADA, RTOS, embedded systems

Architecture Considerations

  • Operational Factors: Availability, resilience, responsiveness, scalability
  • Business Factors: Cost, ease of deployment, risk transference, ease of recovery
  • Technical Factors: Patch availability, inability to patch, power requirements, compute resources

3.2: Security Principles for Enterprise Infrastructure

Learning Objectives

Students will be able to:

  • Apply security principles to device placement and network design
  • Configure network appliances for optimal security
  • Implement secure communication protocols
  • Select effective security controls for enterprise environments

Key Terms

  • Security Zones: Network segments with similar security requirements
  • Attack Surface: Total points where unauthorized access can occur
  • Fail-open: System defaults to allowing access when failure occurs
  • Fail-closed: System defaults to denying access when failure occurs
  • Jump Server: Intermediary server for accessing secure network segments
  • IPS/IDS: Intrusion Prevention/Detection Systems
  • WAF: Web Application Firewall protecting web applications
  • NGFW: Next-Generation Firewall with advanced inspection capabilities

Key Concepts

Infrastructure Considerations

  • Device Placement Strategy: Strategic positioning, data flow consideration, performance impact
  • Security Zones: Network segmentation, DMZ implementation, internal protection
  • Attack Surface Management: Vector identification, asset inventory, continuous monitoring

Network Appliances and Security Devices

  • Access Control Infrastructure: Jump servers, proxy servers, network sensors
  • Intrusion Prevention: IPS/IDS deployment, detection methods, monitoring strategies
  • Load Balancing: Traffic distribution, SSL termination, health checking
  • Firewall Technologies: WAF, UTM, NGFW, Layer 4 vs Layer 7 filtering

  • 3.2.7 Control Selection and Implementation (45 minutes)

3.3: Data Protection Concepts and Strategies

Learning Objectives

Students will be able to:

  • Classify data according to sensitivity and regulatory requirements
  • Implement appropriate data protection methods
  • Apply data security controls based on data states
  • Understand data sovereignty and geolocation implications

Key Terms

  • Data at Rest: Data stored on persistent storage media
  • Data in Transit: Data moving between systems or networks
  • Data in Use: Data being actively processed or accessed
  • Data Sovereignty: Legal concept that data is subject to laws of country where located
  • Encryption: Converting data into unreadable format without decryption key
  • Tokenization: Replacing sensitive data with non-sensitive tokens
  • PII: Personally Identifiable Information
  • PHI: Protected Health Information

Key Concepts

Data Types and Classifications

  • Data Categories: Regulated data, trade secrets, intellectual property, legal information
  • Classification Levels: Public, sensitive, confidential, restricted, private, critical

Data Protection Methods

  • Cryptographic Protection: Encryption, hashing, key management
  • Data Transformation: Masking, tokenization, obfuscation
  • Access Controls: Geographic restrictions, segmentation, permission restrictions

3.4: Resilience and Recovery in Security Architecture

Learning Objectives

Students will be able to:

  • Design high availability systems with appropriate redundancy
  • Implement disaster recovery and business continuity solutions
  • Plan and execute resilience testing procedures
  • Configure backup and recovery systems

Key Terms

  • High Availability (HA): System operational continuity with minimal downtime
  • Hot Site: Fully operational backup facility ready for immediate use
  • Warm Site: Partially configured backup facility requiring setup time
  • Cold Site: Basic backup facility requiring full configuration
  • RTO: Recovery Time Objective - maximum acceptable downtime
  • RPO: Recovery Point Objective - maximum acceptable data loss
  • Failover: Automatic switching to backup system during failure

Key Concepts

High Availability Design

  • Load Balancing vs. Clustering: Traffic distribution, server grouping, failover configurations
  • Redundancy Strategies: Hardware, software, network, and data redundancy

Site Considerations

  • Disaster Recovery Sites: Hot, warm, and cold site implementations
  • Geographic Distribution: Dispersion benefits, regulatory considerations, connectivity

Platform and Service Diversity

  • Platform Diversity: Vendor diversity, technology diversity, OS diversity
  • Multi-cloud Strategies: Provider diversity, hybrid approaches, inter-cloud connectivity
