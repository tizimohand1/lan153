2.1 Compare and contrast common threat actors and motivations ▼
2.1.1 Threat Actors
Nation-State
Government-sponsored actors conducting cyber operations
Characteristics:
  • Extensive resources and funding
  • Highly sophisticated capabilities
  • Long-term strategic objectives
  • Advanced persistent threat (APT) campaigns
Typical Targets: Government agencies, defense contractors, critical infrastructure
Unskilled Attacker
Individuals with limited technical expertise using existing tools
Characteristics:
  • Low skill level, relies on pre-built tools and scripts
  • Limited resources
  • Often called "script kiddies"
  • Opportunistic rather than targeted
Common Tools: Automated vulnerability scanners, downloadable exploit kits
Hacktivist
Individuals or groups using hacking for political or social causes
Characteristics:
  • Moderate technical skills
  • Ideologically motivated
  • Often work in groups or collectives
  • Public campaigns and demonstrations
Common Tactics: Website defacements, DDoS attacks, data leaks
Insider Threat
Trusted individuals within an organization who pose security risks
Types:
  • Malicious insider: Intentionally harmful actions
  • Negligent insider: Unintentional security violations
  • Compromised insider: External actors using legitimate credentials
Risk Factors: Financial stress, job dissatisfaction, ideological conflicts
Organized Crime
Professional criminal organizations conducting cyber operations
Characteristics:
  • Well-funded and organized
  • Profit-driven operations
  • Sophisticated techniques and tools
  • Global networks and coordination
Common Activities: Ransomware operations, financial fraud, identity theft
Shadow IT
Unauthorized IT systems, devices, or services used within organizations
Characteristics:
  • Bypass official IT policies and procedures
  • Often well-intentioned but create security risks
  • Lack proper security controls and monitoring
Examples: Unapproved cloud services, personal devices, unauthorized software
2.1.2 Attributes of Actors
Internal/External
Internal Threats:
  • Employees, contractors, business partners
  • Legitimate access to systems and facilities
  • Insider knowledge of processes and vulnerabilities
External Threats:
  • Attackers from outside the organization
  • Must gain unauthorized access
  • May use social engineering or technical exploits
Resources/Funding
High-Resource Actors: Nation-states, organized crime
Medium-Resource Actors: Hacktivists, some insider threats
Low-Resource Actors: Unskilled attackers, some insider threats
Level of Sophistication/Capability
Advanced: Custom malware, zero-day exploits, APT campaigns
Intermediate: Modified existing tools, social engineering
Basic: Readily available tools, simple attack methods
2.1.3 Motivations
Data Exfiltration
Stealing sensitive information for various reasons
Targets: Intellectual property, customer data, financial records
Espionage
Gathering intelligence for competitive, political, or military advantage
Types: Industrial espionage, state espionage
Service Disruption
Preventing normal operations of services or systems
Methods: DDoS attacks, system sabotage, infrastructure attacks
Financial Gain
Direct monetary profit from cyber activities
Methods: Banking fraud, cryptocurrency theft, ransomware
Philosophical/Political Beliefs
Advancing ideological causes or political agendas
Methods: Website defacements, data leaks, propaganda campaigns
War
Military or strategic advantages in conflicts
Targets: Critical infrastructure, military systems, communications
2.2 Explain common threat vectors and attack surfaces ▼
2.2.1 Message-Based Threats
Email
Attack Types:
  • Phishing attacks: Deceptive emails to steal credentials
  • Malware delivery: Attachments containing malicious code
  • Business Email Compromise (BEC): Targeting executives
  • Email spoofing: Impersonating legitimate senders
Short Message Service (SMS)
Attack Types:
  • SMS phishing (Smishing): Text-based phishing attacks
  • Malware links: URLs leading to malicious downloads
  • Two-factor authentication bypass: SIM swapping attacks
Instant Messaging (IM)
Attack Types:
  • Malware spreading through file transfers and links
  • Social engineering through direct communication
  • Credential harvesting with fake login prompts
2.2.2 Network-Based Threats
Wireless Networks
Vulnerabilities:
  • Open networks: Unencrypted wireless access points
  • Weak encryption: WEP and weak WPA implementations
  • Evil twin attacks: Malicious access points mimicking legitimate ones
  • Man-in-the-middle attacks: Intercepting wireless communications
Bluetooth
Attack Types:
  • Bluejacking: Sending unauthorized messages
  • Bluesnarfing: Unauthorized access to device data
  • Proximity attacks: Short-range exploitation
2.2.3 Supply Chain Attacks
Managed Service Providers (MSPs)
Risks:
  • Third-party access to multiple client systems
  • Privileged access across organizations
  • Cascade effects from single compromise
Vendors
Attack Vectors:
  • Compromised software distribution
  • Modified hardware components
  • Poisoned software updates
2.2.4 Human Vectors/Social Engineering
Phishing
Types:
  • Email phishing: Deceptive emails requesting information
  • Spear phishing: Targeted phishing attacks
  • Whaling: Targeting high-value individuals
  • Clone phishing: Duplicating legitimate emails with malicious modifications
Business Email Compromise
Methods:
  • Executive impersonation: Posing as company leadership
  • Vendor email compromise: Hijacking legitimate communications
  • Invoice fraud: Fake or modified invoices
  • Wire transfer fraud: Unauthorized financial transfers
Pretexting
Creating false scenarios to gather information or gain access
Characteristics:
  • False scenarios: Creating believable but false situations
  • Information gathering: Collecting data for future attacks
  • Trust building: Establishing rapport before exploitation
Watering Hole
Compromising websites frequented by target audience
Strategy:
  • Targeted websites: Compromising sites visited by target audience
  • Strategic placement: Infecting commonly used resources
  • Patient approach: Long-term compromise strategies
Typosquatting
Registering domains with common typos to deceive users
Tactics:
  • Similar domains: Registering domains with common typos
  • User error exploitation: Capitalizing on typing mistakes
  • Credential harvesting: Fake login pages
2.3 Explain various types of vulnerabilities ▼
2.3.1 Application Vulnerabilities
Buffer Overflow
Writing data beyond allocated memory boundaries
Types: Stack overflow, heap overflow
Prevention: Bounds checking, stack canaries, ASLR
Race Conditions
Timing-dependent vulnerabilities in concurrent operations
Components:
  • Time-of-Check (TOC): Verifying conditions at one point
  • Time-of-Use (TOU): Using resources at a different time
  • TOCTOU attacks: Exploiting the gap between check and use
Memory Injection
Exploiting memory management flaws to inject malicious code
Types: Code injection, DLL injection, process hollowing
2.3.2 Web-Based Vulnerabilities
SQL Injection (SQLi)
Injecting malicious SQL commands into web applications
Types: In-band, blind, time-based SQL injection
Impact: Data extraction, database manipulation, authentication bypass
Prevention: Parameterized queries, input validation, least privilege
Cross-Site Scripting (XSS)
Injecting malicious scripts into web pages
Types: Stored XSS, reflected XSS, DOM-based XSS
Impact: Session hijacking, credential theft, malware distribution
Prevention: Input sanitization, output encoding, CSP headers
2.3.3 Hardware Vulnerabilities
Firmware
Low-level software stored in hardware
Vulnerabilities: Backdoors, weak authentication, update mechanisms
Examples: UEFI vulnerabilities, router firmware flaws
End-of-Life
Hardware no longer supported by manufacturer
Risks: No security updates, accumulating vulnerabilities
Legacy
Older hardware still in use but difficult to secure
Mitigation: Network isolation, additional monitoring
2.3.4 Cloud-Specific Vulnerabilities
Common Cloud Vulnerabilities
Risk Areas:
  • Shared responsibility model: Unclear security boundaries
  • Misconfigured storage: Publicly accessible cloud storage
  • Weak access controls: Inadequate identity and access management
  • API vulnerabilities: Insecure cloud service APIs
  • Data residency: Data location and sovereignty issues
2.3.5 Mobile Device Vulnerabilities
Side Loading
Installing applications from unofficial sources
Risks: Malicious app installation, bypassing security controls
Jailbreaking
Removing manufacturer security restrictions
Risks: Disabling security features, installing malicious software
2.4 Given a scenario, analyze indicators of malicious activity ▼
2.4.1 Malware Attacks
Ransomware
Malware that encrypts data and demands payment for decryption
Indicators: File encryption, ransom notes, system lockouts
Variants: Crypto-ransomware, locker ransomware, double extortion
Trojan
Malware disguised as legitimate software
Indicators: Unexpected network connections, performance degradation
Types: Remote access trojans (RATs), banking trojans, downloaders
Worm
Self-replicating malware that spreads across networks
Indicators: Network congestion, rapid spread, system performance issues
Propagation: Network vulnerabilities, removable media, email
Spyware
Malware that secretly monitors and collects information
Indicators: Keystroke logging, screen capture, data transmission
Types: Keyloggers, screen recorders, audio/video capture
Rootkit
Malware that hides its presence and maintains persistence
Indicators: Hidden processes, modified system files, unusual network activity
Types: User-mode rootkits, kernel-mode rootkits, firmware rootkits
Logic Bomb
Malicious code triggered by specific conditions
Triggers: Date/time, user actions, system conditions
2.4.2 Network Attacks
Distributed Denial-of-Service (DDoS)
Overwhelming systems with traffic from multiple sources
Indicators: Service unavailability, network congestion, high resource usage
Types:
  • Amplified: Using protocols that generate larger responses
  • Reflected: Bouncing attack traffic off third-party systems
DNS Attacks
Types: DNS poisoning, DNS tunneling, DNS hijacking
Indicators: Unexpected DNS responses, traffic to malicious domains
On-Path Attacks
Intercepting and potentially modifying network communications
Methods: ARP poisoning, DNS spoofing, BGP hijacking
Indicators: Unexpected network routes, certificate warnings
Credential Replay
Reusing captured authentication credentials
Methods: Session hijacking, pass-the-hash attacks
Indicators: Unusual login patterns, concurrent sessions
2.4.3 Application Attacks
Injection
Types: SQL injection, command injection, LDAP injection
Indicators: Database errors, unexpected command execution
Privilege Escalation
Gaining higher-level access than authorized
Types: Vertical escalation, horizontal escalation
Indicators: Unauthorized access, elevated privileges
Directory Traversal
Accessing files outside intended directories
Indicators: Attempts to access system files, path manipulation
2.4.4 Password Attacks
Password Spraying
Trying common passwords against many accounts
Indicators: Failed logins across multiple accounts
Advantage: Avoiding account lockout policies
Brute Force
Systematically trying all possible password combinations
Indicators: High volume of failed login attempts
Types: Dictionary attacks, rainbow table attacks
2.4.5 Activity Indicators
Account Lockout
Multiple failed authentication attempts
Investigation: Review login logs, check for attack patterns
Concurrent Session Usage
Same account accessed from multiple locations simultaneously
Analysis: Geographic analysis, device fingerprinting
Impossible Travel
Account access from geographically impossible locations
Analysis: Time stamps, geographic distance, travel time
Resource Consumption
Unusual system resource usage patterns
Monitoring: CPU, memory, disk, network utilization
Out-of-Cycle Logging
Log entries occurring outside normal patterns
Analysis: Time-based patterns, user behavior, system cycles
Missing Logs
Expected log entries that are absent
Causes: Log tampering, system failures, configuration issues
2.5 Explain the purpose of mitigation techniques used to secure the enterprise ▼
2.5.1 Network Security Controls
Segmentation
Isolating network areas to contain threats and limit access
Implementation: VLANs, subnets, firewalls, air gaps
Benefits: Threat containment, reduced attack surface, compliance
Isolation
Separating systems or processes to prevent contamination
Types: Physical isolation, logical isolation, process isolation
Implementation: Air gaps, VMs, containers, sandboxes
2.5.2 Access Control
Access Control List (ACL)
Defining specific permissions for resources
Components: Subjects, objects, permissions
Implementation: File systems, network devices, applications
Permissions
Types: Read, write, execute, delete, modify
Models: Discretionary (DAC), mandatory (MAC), role-based (RBAC)
2.5.3 Application Security
Application Allow List
Permitting only approved applications to run
Implementation: Whitelist-based application control
Benefits: Malware prevention, compliance enforcement
2.5.4 System Maintenance
Patching
Fixing known vulnerabilities in software and systems
Process: Testing, staging, deployment, verification
Management: Patch cycles, emergency patches, rollback procedures
