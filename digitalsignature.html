Digital Signatures & X.509 Certificates

1. Core Concepts

In asymmetric cryptography, keys work in matching pairs. To prove your identity online and guarantee your data hasn't been altered, you use a Digital Signature.

The Mechanics: Identity & Integrity

  • Integrity (Hashing): The original message is processed through a cryptographic hash function, creating a unique fixed-length mathematical fingerprint. If even one bit of the message changes, the hash changes completely.
  • Identity (Signing): The sender encrypts that hash using their private key. Because only the sender possesses that private key, this action establishes non-repudiation. Anyone with the corresponding public key can decrypt it to verify the sender's identity.
The Verification Formula:
The recipient decrypts the received signature using the sender's public key to reveal the original hash. They then re-hash the received message. If Hash A == Hash B, identity and integrity are proven!

2. What is an X.509 Certificate?

A digital signature proves who signed a message only if you can trust that the public key belongs to them. An X.509 Certificate solves this problem. It acts as a digital driver's license, bound to an identity by a trusted third party called a Certificate Authority (CA).
