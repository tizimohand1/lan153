What Security Operations Covers

Security Operations focuses on the day-to-day activities that security professionals perform to protect organizational assets. Think of this as the "hands-on" domain where theory meets practice.

Security Operations is like running a hospital emergency room. You need established procedures (baselines), proper equipment (hardened systems), constant monitoring (alerting), quick response to incidents, and detailed documentation of everything that happens.

4.1 Common Security Techniques for Computing Resources

This is the most heavily tested sub-objective in Domain 4. Expect 6-7 questions on this topic alone!

Secure Baselines

A secure baseline is like a master recipe that ensures every system is configured consistently and securely.

Think of secure baselines like building codes for houses. Just as building codes ensure all houses meet minimum safety standards, secure baselines ensure all systems meet minimum security standards.

Baseline Lifecycle:

  • Establish: Create the standard configuration
  • Deploy: Apply the baseline to systems
  • Maintain: Keep baselines current and detect drift

Hardening Targets

System hardening is like installing security features in a building - you remove unnecessary access points and strengthen what remains.

Key Hardening Targets for Exam:

  • Mobile Devices: Encryption, strong authentication, app vetting, remote wipe
  • Workstations: Antivirus, host firewall, UAC, screen locks, FDE
  • Network Infrastructure: Switches (port security, VLAN), Routers (ACLs, disable unused services)
  • Cloud Infrastructure: IAM, security groups, encryption, monitoring
  • Servers: Minimal installation, service management, file permissions
  • Specialized Systems: ICS/SCADA, embedded systems, RTOS, IoT

Mobile Solutions - HIGH EXAM FOCUS

Mobile deployment models are heavily tested. Know the differences between BYOD, COPE, and CYOD!

Mobile Deployment Models:

  • BYOD (Bring Your Own Device): Employee-owned, personal use allowed, organization provides access
  • COPE (Corporate-Owned, Personally Enabled): Company-owned, personal use allowed
  • CYOD (Choose Your Own Device): Company provides approved device list for employee selection

Mobile Connection Methods:

  • Cellular: 4G/5G networks, encrypted by default but use VPN for corporate access
  • Wi-Fi: Use WPA3, verify certificates, implement VPN
  • Bluetooth: Disable when not needed, use pairing, monitor connections

Wireless Security Settings

Critical Wireless Protocols:

  • WPA3: Latest Wi-Fi security, 128-bit personal/192-bit enterprise
  • AAA/RADIUS: Centralized authentication for enterprise networks
  • EAP Variants: EAP-TLS (certificates), PEAP (tunneled), EAP-TTLS

Application Security

Application Security Techniques:

  • Input Validation: Whitelist approach, check data types and length
  • Secure Cookies: Secure flag (HTTPS only), HttpOnly (no JavaScript access)
  • Static Code Analysis: Examine source code for vulnerabilities
  • Code Signing: Digital signatures to verify software authenticity
  • Sandboxing: Isolate applications in controlled environments

4.2 Hardware, Software, and Data Asset Management

Asset Management Lifecycle

Asset management is like managing a library. You need to know what books you have (inventory), who has them (assignment), where they are (tracking), and when to remove old ones (disposal).

Asset Management Process:

  • Acquisition/Procurement: Secure purchasing processes, vendor validation
  • Assignment/Accounting: Track ownership and classification
  • Monitoring/Asset Tracking: Inventory management and enumeration
  • Disposal/Decommissioning: Secure sanitization, destruction, certification

Data Sanitization and Disposal

Know the difference between sanitization (making data unrecoverable) and destruction (physical destruction of media).

Disposal Methods:

  • Sanitization: Overwriting data multiple times to make it unrecoverable
  • Destruction: Physical destruction of storage media
  • Certification: Documentation proving proper disposal
  • Data Retention: Policies for how long to keep different types of data

4.3 Vulnerability Management Activities

Vulnerability management is a continuous process. Remember the cycle: Identify → Analyze → Respond → Validate → Report

Identification Methods

Vulnerability Discovery:

  • Vulnerability Scans: Automated tools that check for known vulnerabilities
  • Application Security: Static analysis (code review), Dynamic analysis (running tests)
  • Threat Feeds: OSINT, proprietary feeds, information sharing organizations
  • Penetration Testing: Authorized simulated attacks
  • Bug Bounty Programs: Crowdsourced vulnerability discovery

Analysis and Prioritization

Vulnerability Analysis:

  • CVSS (Common Vulnerability Scoring System): 0-10 scale for vulnerability severity
  • CVE (Common Vulnerability Enumeration): Unique identifiers for vulnerabilities
  • False Positives/Negatives: Incorrect scanner results that need validation
  • Environmental Variables: How vulnerability impacts your specific environment

Response and Remediation

Remediation Options:

  • Patching: Apply vendor-provided fixes
  • Segmentation: Isolate vulnerable systems
  • Compensating Controls: Alternative security measures
  • Insurance: Transfer financial risk
  • Exceptions/Exemptions: Documented acceptance of risk

4.4 Security Alerting and Monitoring

Monitoring Computing Resources

Security monitoring is like having security cameras throughout a building. You need to watch systems, applications, and infrastructure constantly to detect problems early.

Monitoring Activities:

  • Log Aggregation: Collecting logs from multiple sources
  • Alerting: Automated notifications of security events
  • Scanning: Regular vulnerability and compliance checks
  • Reporting: Regular summaries of security posture
  • Alert Response: Quarantine, tuning, validation

Monitoring Tools

Essential Monitoring Tools:

  • SIEM: Security Information and Event Management - central log analysis
  • SCAP: Security Content Automation Protocol - standardized vulnerability data
  • DLP: Data Loss Prevention - monitors data movement
  • SNMP Traps: Network device notifications
  • NetFlow: Network traffic analysis
  • Vulnerability Scanners: Automated vulnerability detection

4.5 Enterprise Security Enhancement

Firewall Configuration

Firewall Management:

  • Rules: Allow/deny traffic based on criteria
  • Access Lists: Ordered list of permit/deny statements
  • Ports/Protocols: Specify what traffic types are allowed
  • Screened Subnets (DMZ): Isolated network segments

IDS/IPS Implementation

Intrusion Detection/Prevention:

  • Signature-based: Detect known attack patterns
  • Anomaly-based: Detect deviations from normal behavior
  • Trends: Analyze patterns over time

Email Security

Know the email security acronyms: SPF, DKIM, DMARC - these appear frequently on the exam!

Email Protection:

  • SPF (Sender Policy Framework): Validates sending IP addresses
  • DKIM (DomainKeys Identified Mail): Digital signatures for email
  • DMARC: Policy for handling SPF/DKIM failures
  • Email Gateway: Centralized email security filtering

4.6 Identity and Access Management

IAM is fundamental to security. Focus on access control models, MFA implementation, and privileged access management.

Access Control Models

Access Control Types:

  • Mandatory Access Control (MAC): System-enforced, based on security labels
  • Discretionary Access Control (DAC): Owner controls access
  • Role-Based Access Control (RBAC): Access based on job roles
  • Rule-Based Access Control: Access based on rules and conditions
  • Attribute-Based Access Control (ABAC): Dynamic access based on attributes

Multifactor Authentication

Authentication Factors:

  • Something You Know: Passwords, PINs
  • Something You Have: Tokens, smart cards, mobile devices
  • Something You Are: Biometrics (fingerprint, retina)
  • Somewhere You Are: Location-based authentication

Single Sign-On (SSO)

SSO Technologies:

  • LDAP: Directory services protocol
  • OAuth: Authorization framework
  • SAML: Security Assertions Markup Language

Privileged Access Management

PAM Components:

  • Just-in-Time Permissions: Temporary elevated access
  • Password Vaulting: Secure storage of privileged passwords
  • Ephemeral Credentials: Short-lived authentication tokens

4.7 Automation and Orchestration

Use Cases for Automation

Automation Applications:

  • User Provisioning: Automated account creation/deletion
  • Resource Provisioning: Automated infrastructure deployment
  • Guard Rails: Automated policy enforcement
  • Security Groups: Automated firewall rule management
  • Ticket Creation: Automated incident tracking
  • Escalation: Automated alert escalation processes
  • Service Management: Automated enabling/disabling of services
  • CI/CD Integration: Security testing in development pipelines

Benefits and Considerations

Automation Benefits:

  • Efficiency: Faster response times and reduced manual work
  • Consistency: Standardized configurations and processes
  • Scaling: Handle larger environments with same staff
  • Employee Retention: Reduce repetitive tasks
  • Workforce Multiplier: Amplify human capabilities

Automation Challenges:

  • Complexity: More moving parts to manage
  • Cost: Initial setup and ongoing maintenance
  • Single Point of Failure: Automation system dependencies
  • Technical Debt: Maintaining automation code
  • Ongoing Support: Need skilled staff to maintain

4.8 Incident Response Activities

Memorize the incident response process phases. They appear frequently in scenario questions!

Incident Response Process

Incident response is like emergency medical response. You need preparation (training, procedures), detection (monitoring), analysis (diagnosis), containment (stopping the spread), eradication (removing the cause), recovery (returning to normal), and lessons learned (improving for next time).

IR Process Phases:

  • 1. Preparation: Policies, procedures, training, tools
  • 2. Detection: Identifying potential security incidents
  • 3. Analysis: Determining scope and impact
  • 4. Containment: Stopping incident spread
  • 5. Eradication: Removing the root cause
  • 6. Recovery: Restoring normal operations
  • 7. Lessons Learned: Post-incident review and improvement

Training and Testing

IR Preparedness:

  • Tabletop Exercises: Discussion-based incident scenarios
  • Simulations: Hands-on practice with realistic scenarios
  • Root Cause Analysis: Determining why incidents occurred
  • Threat Hunting: Proactive searching for threats

Digital Forensics

Forensics Process:

  • Legal Hold: Preserving evidence for legal proceedings
  • Chain of Custody: Documenting evidence handling
  • Acquisition: Creating forensic copies of evidence
  • Preservation: Maintaining evidence integrity
  • E-discovery: Electronic evidence identification and collection

4.9 Data Sources for Investigation

Log Data Sources

Critical Log Types:

  • Firewall Logs: Network traffic allow/deny decisions
  • Application Logs: Software-specific events and errors
  • Endpoint Logs: Workstation and server activities
  • OS Security Logs: Authentication and authorization events
  • IPS/IDS Logs: Intrusion detection and prevention events
  • Network Logs: Network device and traffic information
  • Metadata: Data about data (file creation times, etc.)

Investigation Data Sources

Additional Investigation Resources:

  • Vulnerability Scans: Historical vulnerability data
  • Automated Reports: Scheduled security summaries
  • Dashboards: Real-time security metrics visualization
  • Packet Captures: Raw network traffic analysis

Practice Questions - Domain 4

These practice questions mirror the style and difficulty of actual Security+ exam questions. Focus on scenario-based thinking!
1. Your organization is implementing a mobile device strategy where employees can choose from a pre-approved list of smartphones and tablets that the company will purchase. What deployment model is this?
  • A) BYOD (Bring Your Own Device)
  • B) COPE (Corporate-Owned, Personally Enabled)
  • C) CYOD (Choose Your Own Device)
  • D) MDM (Mobile Device Management)
Correct Answer: C) CYOD
CYOD allows employees to choose from a company-approved list of devices that the organization purchases. BYOD uses employee-owned devices, COPE is company-owned with personal use allowed, and MDM is a management technology, not a deployment model.
2. During a vulnerability assessment, you discover a critical vulnerability with a CVSS score of 9.2. However, the affected system is air-gapped and not accessible from the internet. What should be your primary consideration when prioritizing remediation?
  • A) The CVSS score alone determines priority
  • B) Environmental variables and exposure factor
  • C) Always patch critical vulnerabilities immediately
  • D) Air-gapped systems don't need patching
Correct Answer: B) Environmental variables and exposure factor
While CVSS provides a baseline severity score, environmental variables (like network isolation) and exposure factors significantly impact actual risk. An air-gapped system with limited access may not require immediate patching despite a high CVSS score.
3. What is the primary difference between WPA2 and WPA3 wireless security protocols?
  • A) WPA3 uses stronger passwords
  • B) WPA3 provides protection against offline dictionary attacks
  • C) WPA3 requires enterprise certificates
  • D) WPA3 only works with newer hardware
Correct Answer: B) WPA3 provides protection against offline dictionary attacks
WPA3's key improvement is Simultaneous Authentication of Equals (SAE), which replaces WPA2's Pre-Shared Key (PSK) method and provides protection against offline dictionary attacks, even if an attacker captures the handshake.
4. In the incident response process, what phase comes immediately after "Analysis"?
  • A) Detection
  • B) Eradication
  • C) Containment
  • D) Recovery
Correct Answer: C) Containment
The IR process follows this order: Preparation → Detection → Analysis → Containment → Eradication → Recovery → Lessons Learned. After analyzing the incident to understand its scope and impact, the next step is to contain it to prevent further damage.
5. Which email security protocol validates that emails are sent from authorized IP addresses for a domain?
  • A) SPF (Sender Policy Framework)
  • B) DKIM (DomainKeys Identified Mail)
  • C) DMARC (Domain Message Authentication Reporting and Conformance)
  • D) S/MIME (Secure/Multipurpose Internet Mail Extensions)
Correct Answer: A) SPF
SPF validates that emails are sent from authorized IP addresses for a domain. DKIM provides digital signatures for email authenticity, DMARC sets policies for handling SPF/DKIM failures, and S/MIME provides end-to-end email encryption.
6. What type of access control model uses security labels and is typically found in government and military environments?
  • A) Mandatory Access Control (MAC)
  • B) Discretionary Access Control (DAC)
  • C) Role-Based Access Control (RBAC)
  • D) Attribute-Based Access Control (ABAC)
Correct Answer: A) MAC
Mandatory Access Control uses security labels (like Top Secret, Secret, Confidential) and is system-enforced rather than user-controlled. It's commonly used in government and military environments where strict information classification is required.
